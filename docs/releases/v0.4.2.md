# Lumi Social Intelligence v0.4.2

**Release type:** care release<br>
**Public status:** public-safe contract/evidence release<br>
**Core principle:** Live Surface reduces uncertainty before it reduces task time.

v0.4.2 fixes an important missed part of the original Live Surface release: small safe actions, especially emoji reactions, must feel instant and predictable. A user should not have to watch visible tool searches, database archaeology, or multi-step uncertainty for an already-warmed action like “add emoji to my last message.”

## What changed

### Instant Live Surface reaction contract

For reaction requests such as “Add emoji to my last message,” v0.4.2 defines the public-safe host contract:

- intent: `react_to_last_user_message`
- execution lane: `instant_control_plane`
- visible latency budget: `instant`
- agent tool calls allowed: `0 tool calls`
- tool discovery allowed: `false`
- DB archaeology allowed: `false`
- message-id guessing allowed: `false` — the system must not guess
- target resolution: `host_envelope_required`

If the host envelope does not already expose a safe last-message target, the system must fail closed:

> I can’t safely find the last message id, so I won’t guess.

### Next-Step Care

v0.4.2 names **Next-Step Care** as care infrastructure, not polish. Before side effects, the system should reduce uncertainty with tiny predictable narration unless the action is so small and safe that instant action is kinder.

Examples:

- Before action: “I’m going to react with ❤️ to your latest Telegram message.”
- After success: “Done ❤️”
- Fail closed: “I can’t safely find the last message id, so I won’t guess.”

This matters for neurodivergent users because predictable next steps lower cognitive load and make powerful invisible systems feel safer.

## Public boundary

This repository still does **not** ship private Hermes runtime code, raw Telegram state, chat IDs, or message IDs. It documents and tests the contract that a host runtime can implement.
