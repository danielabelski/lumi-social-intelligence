# Lumi Social Intelligence v0.4.0: review-gated Live Surface natural-language controls

v0.4.0 is the real public release of review-gated Live Surface natural-language controls.

This is not a shadow-only release. The public API now produces review-gated control cards for safe session readiness state and fails closed when a requested personal-data surface is not available.

## Release notes

Live Surface is the capability boundary between “the assistant has safe, warmed context” and “the assistant is allowed to act.” v0.4.0 turns that boundary into a small public API: ordinary user phrasing can be parsed into reviewable host-readiness state, while personal-data surfaces and real-world side effects remain behind explicit review.

In short: Live Surface separates **being ready** from **being allowed to act**.

### Added

- Real review-gated Live Surface public API via `apply_review_gated_control`.
- Semantic intent parsing for ordinary phrases such as keeping a safe surface ready, surfacing it only when relevant, or requesting that a missing surface be added later.
- Host-applyable review cards for safe session-readiness state.
- Public evidence artifacts for the v0.4.0 Live Surface path.
- Release-gate coverage for the zero-write safety counters and public claim boundary.

### Changed

- Public wording now presents the feature as **Live Surface** rather than the older longer controls-based label.
- The release story now distinguishes warmed context from permission to act more explicitly.
- Evidence wording now frames v0.4.0 as a real review-gated public capability, not a shadow-only preview.

### Safety boundary

- Safe session-readiness state can be represented by the host public API.
- Personal-data requests fail closed until the surface is explicitly added and reviewed.
- External sends, durable memory writes, scheduler/config changes, and runtime mutation are not granted by Live Surface readiness.

## Claim boundary

This release claims real review-gated public Live Surface. It is not autonomous private runtime automation.

## Not claimed

- Telegram sends or reactions
- Calendar/email reads without explicit reviewed surface membership
- Runtime config or scheduler mutation
- Durable memory writes
- Raw private Hermes runtime state
